Introduction

This example is to show the user how to use iSIGHT to match a basic model. There is a Best Practice on detailed use of iSIGHT which can be referred to at any time.

From time to time throughout the tutorial, you will be referred to the iSIGHT Best Practice for more detailed information on a topic. You will be able to return to the original page using the return button at the foot of the Best Practice page.

Please read the iSIGHT introduction page now, then return to this page to continue with the exercise.

Set up model

The method described below is recommended for ANY matching exercise, regardless of whether an optimiser is to be used.

Read the Best Practice page on setting up the sc03 model and then return to this page to continue with the exercise.

  1. In the SC03 general example base model, ensure that user parameters are added to the user variables as follows:

 

Name Expression
fcsh23 1
fcsh24 1
fcsh25 1
fcsh26 1
fcsh37 1
fcsh50 1
fcsh61 1
fcsh62 1
fcvh05 1
fcdh08 1

 

  1. Define the heat transfer expressions in the following boundary conditions as shown. These are the factors which are to be varied as part of the matching exercise with the additional term to represent the factor.

 

Boundary Condition New htc expression
stream 23 fcsh23*FRD(NH)
stream 24 fcsh24*FRD(NH)
stream 25 fcsh25*FRD(NH)
stream 26 fcsh26*FRD(NH)
stream 37 fcsh37*FRD(NH-NI)
stream 50 fcsh50*MFCD(2250.3/102,5.3,17.6,SM50/102)
stream 61 fcsh61*FRD(NH)
stream 62 fcsh62*FRD(NH)
void 5.1 fcvh05*NVP(190)
duct 8 fcdh08*FRD(NH-NI)

 

  1. Produce key values in SC03 which record the difference between measured and predicted data.    

    e.g.        DIFFP1 = tk@P1-GRP1(t)
    where :   GRP1(t) is the measured data at that particular point
                 tk@P1 is the predicted metal temperature at position P1

     

  2. Save the model (File/Save file/Database)

 

Create iSIGHT files ready for matching

A new facility now exists to automate the production of the iSIGHT input file. Follow the steps below:

  1. Load the SC77 plugin (File/Extensions). Select the PROD toggle and them hit the LOAD button. Select the file called libsc77.so and then OK to exit from the panel.

  2. Select Analysis/iSIGHT Gen, a panel should then appear:

  1. Select the SC03 Inputs button, the following information panel should then appear:

OK this panel. The SC03 inputs panel will appear.

Ensure that this panel only contains the user expressions for the factors that you wish to change (as set in the previous section)

Note that you will need to use the More button to see all the data

Type 0.1 into the Lower field, 1 into the Current field and 3 into the Upper field for all user expressions

OK the panel

  1. Select the SC03 Outputs button. The following information panel should then appear:

OK this panel, and then the SC03 Output panel will appear.

Ensure that this panel only contains the key values that refer to the difference plots (as set in the previous section)

Note that you will need to use the More button to see all the data

OK the panel

  1. Select the Task Plans button

Select the NLPQL toggle and set the number of iterations to 40

OK the panel

  1. Select the Objective Function button. The following panel should then appear

 

Simply OK the panel

  1. Select the Write iSIGHT Files button. The following panel should then appear:

Type in a name for the iSIGHT description file

OK the panel. The following then appears:

OK the panel. The following then appears:

Type in the name of the SC03 .pm file

OK the panel. The following then appears:

A text file called sum_error.awk will have been produced in the directory that you have run SC03 in. Use a text editor to modify all the weighting factors to 1, i.e.

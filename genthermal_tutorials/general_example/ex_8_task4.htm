Exercise 8 - Task 4: Windage Heating in 'Rectangular' Cavities

Similar to the labyrinth seal, viscous forces in the boundary conditions of rectangular cavities can result in significant heat inputs to the flow. The RECWND correlation works in a similar way to the LAB43231 correlation to calculate the power input to the air as a result of these viscous forces.

Instructions

The first thing to point out is that although the correlation is for rectangular cavities, there are very few cavities in a gas turbine engine that are actually rectangular! As long as the cavity has two sides, a top and a bottom then the RECWND correlation can be used. If we look again at the Trent 1000 HP turbine/fixed structure, there are a couple of places where the RECWND correlation can be used (highlighted red):

Figure 14 - Rectangular cavities between Trent 1000 HP turbine and static structure

In the general example model, there are a couple of cavities that are very similar to these. The first one is modelled as duct 2, and takes its inputs from the first labyrinth seal - duct 1 and stream 25:

Figure 15 - Rectangular cavities in General Example model

The input panel for duct 2 is shown below in Figure 16. This looks very similar to the input panel for the labyrinth seal. The mass flow into the duct is equal to the total mass flow through the labyrinth seal, W2L101 (this could also be expressed as SM25 + DM1). The inlet temperature is simply uses the MIX( ) function to combine the flow from stream 25 with the flow from duct 1. Both of these temperatures are in the absolute frame of reference, so they can be used directly in the MIX( ) function.

Figure 16 - Duct 2 input panel

As with the labyrinth seal, convention puts duct side A on the rotating component, and side B on the static component. The RECWND correlation is used in the Heat Pickup field for side A of the duct, and it requires 9 input parameters to perform its calculations:

RECWND(Tabs in, wdisc1, wdisc2, wouter, winner, mass flow, inlet swirl velocity, Area, Geometry index)

The entire expression from the heat pickup field above is shown below:

RECWND(MIX(25,-1),NH,0,NH,NH,W2L101,DV1,DA2a,1)

These are similar to the parameters required for the LAB43231 correlation, except the rotational speeds of the top and bottom of the cavity are required (wouter & winner). It does not matter which side of the rectangle is specified as disc 1 and which is disc 2, as long as the convention is carried over into the geometry input panel. As with the labyrinth seal correlation, the geometry input panel can be found in SC03 on the Thermals menu, then RECWND. The geometry panel for duct 2 is shown below in Figure 17:

 Figure 17 - Duct 2 RECWND geometry input panel

The top four boxes are easy to define using the Query Geometry tool to extract the required data from the model. The roughness and moment factor fields should be left as 0 and 1 respectively unless otherwise specified in the design scheme. The inner and outer radii of integration tell SC03 how much of the cavity to calculate the heat input for. When the cavity is of a fairly constant width, and there are no inputs to or outputs from the cavity along the boundary condition length, then these values should be equal to the inner and outer shroud radii. The cavity width should be an average width of the entire cavity.

However, if the width of the cavity changes by a large amount, or there is an additional input to the cavity part way up the cavity, or the heat transfer coefficient changes along the cavity, then the radii of integration will not necessarily be equal to the inner and outer shroud radii. For example, consider a cavity with an inner radius of 100mm and an outer radius of 200mm with a constant width of 25mm. Halfway up the cavity  i.e. at a radius of 150mm, there is a stream flowing into the cavity. This would be modelled as two separate ducts - one going from the cavity inlet up to 150mm, and another from 150mm to 200mm with an additional mass flow and power input. The size of the cavity has not changed - the inner radius is still 100mm and the outer radius 200mm, but the first duct would have an inner radius of integration of 100mm and an outer radius of integration of 150mm. The second duct would have an inner radius of integration of 150mm and an outer radius of 200mm.

The swirl velocity field contains a user expression similar to the labyrinth seal that varies the swirl velocity linearly through the cavity. As with the LAB43231 correlation, there is a separate correlation for calculating the swirl velocity in a rectangular cavity - RECSV. As with SV43231, RECSV requires all of the same input parameters as RECWND apart from the area.

There is an issue with the RECWND and RECSV correlations where SC03 will fail to run if the velocities of both discs are zero at any point in the cycle. This usually occurs at t=0s. this can be fixed by editing the cycle definition so that the velocities (NH, NI, and NL) are set to a small value at t=0 i.e. 1rad/s.

Use the method described above to apply the boundary conditions to the second rotor/stator cavity including windage heating calculated using the RECWND correlation (define boundary condition as duct 4 to maintain interdependencies of subsequent boundary conditions)

  • Use the Query Geometry tool to extract the data required for the RECWND geometry panel

  • Mass flow into the cavity is equal to the total mass flow through seal 2

  • The pressure in the cavity is equal to P2C11

  • Select an appropriate heat transfer correlation

  • Define RECSWIRL2 user parameter using the RECSV correlation

Running the SC03 model

Once all of the four tasks have been completed, it should be possible to run the model. Solutions for all of the tasks are given in the Appendix to check your model against. It is not essential that all of the data is the same as the solutions. For example, scaling factors on heat transfer coefficients are a matter of judgement in this example. As mentioned previously, these would be determined by matching the model to test data. Also, some of the geometry data required by the LAB43231 and RECWND correlations again comes down to judgement. As long as the values used are similar then the model results should be largely the same. Figure 20 below shows a contour plot of metal temperature in K at the end of the high power condition (t=4000s).

If there is time, try varying some of the swirl velocities, either by changing the swirl ratio used in the correlations or in the Swirl velocity field.

Note:
When SC03 tries to write out the Summary File after a run, it will return an error relating to the SWIRL1 user parameter. This is not an error in the model, but a problem with SC03 version 10E0 trying to put a single value into the summary file for a parameter that varies with time.

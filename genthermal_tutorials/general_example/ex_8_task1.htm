Exercise 8 - Task 1: ABS2REL and REL2ABS correlations

The first task explains how the ABS2REL and REL2ABS correlations within the SC77 plug-in can be used to change the frame of reference - Absolute for static components, Relative for rotating components.

Updated General Example model

The updated General Example model required for these tasks can be found using a UNIX workstation in the following location:

/usr/rreng/data/sc77/general.example.ABSREL.pm

If you cannot find the file at this location, it is also available on the web.

This model will not run as there are several boundary conditions missing. Once all four tasks have been completed, it should be possible to run the SC03 model.

Instructions

Unless told otherwise, SC03 will assume that the temperature defined in the boundary condition dialogue box is the correct one - absolute on a static component, relative on a rotating component. As shown in the theory section, if the temperature used is not in the correct frame, the air temperatures used by SC03 can be wrong. There are two ways in SC03 of ensuring that the correct temperature is used. The first is to ensure that the correct temperature is specified in the first place. 

The equations discussed in the Theory section show that to convert from Absolute to Relative Total temperatures and back again requires the Swirl velocity, Vf to be known. As this is something that will vary with time, it is not practical to manually convert the temperature into the correct frame of reference. The SC77 plug-in contains two correlations that take a number of inputs and converts either an Absolute Total temperature into a Relative Total temperature, or vice versa. These are the ABS2REL and REL2ABS correlations. The correlation is used in the Inlet Temperature field of either a Stream or Duct input panel, and requires 5 input parameters:

ABS2REL(Tabs in, Radius, wstator, wrotor, b)

The radius must be specified as the radius at which the conversion is taking place, it cannot be specified as the variable 'r' or 'y' but must be defined as a number. For the REL2ABS conversion, the input parameters are the same, except that the input temperature should be a relative total temperature, not an absolute total temperature.

If this correlation is used, it only needs to be used once per system i.e. when the flow jumps from a static component to a rotating component. Using interdependencies ensures that subsequent streams use the relative total temperature. When the flow jumps from a rotating component on to a static component, the REL2ABS correlation should be used to change the temperature back into an absolute total temperature.

In the general.example.ABSREL.pm model, the ABS2REL functionality is used when the flow jumps from the static structure in front of the compressor (Stream 6), onto the first HP compressor disc shown in Figure 2 below (Stream 1, Stream 36 and Stream 76):

Figure 2 - HP Compressor drum from General Example model

The input panel for Stream 36 is shown below in Figure 3. In this scenario, the flow from stream 6 is split between stream 1, stream 36 and steam 76. Some of the flow goes into stream 1, flowing up the front face of the first disc, and the remainder is split 50:50 between stream 36 and stream 76. 

Figure 3 - Stream 36 Input Panel

The ABS2REL correlation is used to convert the Absolute Total temperature at the exit of stream 6 into a Relative Total temperature. The radius of the disc bore at the start of steam 6 is 160mm, stream 6 is flowing down a stationary component, so the rotational speed of the stator is 0, and the disc is rotating at HP shaft speed, NH. The swirl ratio at the entry to the stream is defined as 0.3, that is, the air is rotating at 0.3 times disc speed. Note that when using the ABS2REL or REL2ABS correlation, the Swirl Velocity field in the input panel is left as 0.

The pressure in this region is defined as P1C06. This is a parameter stored in the cycle definition. Normally, the cycle definition contains pressures, temperatures and flows for a number of key locations, or stations, throughout the engine (P26, P30, P44, etc). Pressures for cavities in between these stations are defined either by adding data to the cycle definition (the general example model defines most of its pressures in this way), or using scaling expressions that are derived from an Air Systems SPAN model. These are normally stored as user parameters (this allows a pressure to be defined as a function of one or more pressure from the cycle definition - see Pressure & Temperature scaling on Thermo-mechanics web). The user parameters P2C10 - P2C12 in the general example model have been defined in this way.

The heat transfer coefficient uses the FRD correlation, which applies to a free disc with turbulent flow over its surface. More details of all of the heat transfer correlations used in SC03 can be found on the Heat Transfer & Windage Correlations section of the Thermo-mechanics web. The only required input for this correlation is the rotational speed of the component. A factor of 0.8 has been applied to this correlation. These factors are usually derived from a matching exercise similar to that done in the General Example exercise 5.

Using the information and the example above, add the stream 76 boundary condition to the general.example.ABSREL.pm model.

  • Assume Swirl ratio is the same as for stream 36

  • Remember mass flow is split 50:50 between stream 36 and stream 76

  • Assume the stream 76 has the same heat transfer coefficient as stream 36

Stream 9 and stream 11 are also missing from the SC03 model. The total mass flow into stream 9 and stream 11 is defined in the cycle definition as W1L027, and the inlet absolute total temperature as T1C07. Add streams 9 and 11 to the SC03 model.

  • Assume the mass flow is split 50:50 between stream 9 and stream 11

  • Assume a swirl ratio, b, of 0.4

  • Assume an HTC of 0.7 times free disc

  • Remember that stream 11 is on the IP shaft

Exercise 6 - Matching using iSIGHT

  • Set up model

  • Create iSIGHT files

  • Start iSIGHT

  • Evaluate

 

Start using iSIGHT

The description file that has now been set up can be read into iSIGHT.

Note - This tutorial has been written for iSIGHT version 5.0. The interface is considerably different from iSIGHT version 4.0.

Note that iSIGHT will only run on a Sun platform.

  1. In a UNIX shell type:

isight

Note – there is only one licence for iSIGHT within the Thermal Systems community. If it is already being used then the following message will appear :

Maximum # users for "isight_pro" reached

If you can’t access iSIGHT make sure that your .cshrc.personal file contains the path to iSIGHT version 5.0:

set path = ($path /usr/trials/iSIGHT/bin)

On entering iSIGHT, the Task Manager panel is displayed

  1. Use File/Open and select the .desc file that you have just created from SC03.  

  1. There is one final parameter to modify before proceeding with the optimisation.
    From the Task Manager panel, select the Task Plan icon. The iSIGHT problem definition panel should then appear.

Double click the text Optimisation: SC77 so that the iSIGHT optimisation panel appears.

Select the Step Details tab and change these details: 

Termination Accuracy = 0.0001
Relative Gradient Step = 0.01

OK the panels until you return to the Task Manager panel.

 

Monitor 

Please refer to the Best Practice Monitor Page for detailed help then return to this page to continue with the exercise.

  1. Select the Monitor button. This shows the status of the optimisation.
    It is suggested that the following are displayed

  1. Graph of objective: Insert/Graph/Parameters/Objective/Add

  2. Click OK

  3. Table of variables: Insert/Table/Type/Inputs

  4. Click OK
    Leave the solution monitor panel open

 

Execute

Please refer to the Best Practice Execute Page for detailed help then return to this page to continue with the exercise.

  1. In the Task Manager panel click the Log button to open the Message Log file

  2. Detach the window using the DETACH button.

  3. During the optimisation look for errors appearing in this window in yellow. There shouldn't be any!

  4. Under Execution details hit the Execute icon. The job should now run. You will be able to see in the Log window comments to show when the sc03 runs are started and ended.
    Note that the Run Mode has been set to single run (to check that everything is working OK). The task plan can be turned on when a successful single run has been completed.
    Thus, once this has worked, use the drop down listing to change the single to task plan

  5. When the job starts, iSIGHT will do a couple of runs before writing to the objective window. 

  6. The current value of the factors for each run will appear in the solution monitor window and will be updated at the start of each run.
    The HISTORY directory stores the file containing the user parameters for each run (it is easiest to list them in time order to pick out the most recent one).
    In the main iSIGHT directory there will be files named GR****.item, where **** = the graph name defined by you. These are the difference plot graphs specified by you in the sc03 model. These graphs can be read in and viewed in the normal way in sc03 at any time.

  7. When the Objectives line becomes flat, or iSIGHT completes the task, the answers must be viewed to check they are sensible and that the optimisation was valid.

 

Evaluate

When the optimisation is complete:

  • Check that none of the factors have risen to the upper or dropped to the lower limit. If they have, the limits should be altered. If they need to be altered, so as to make the value outside the limits, it is possible that the physics is wrong.

  • Check that the factors for any two consecutive streams are similar. If they are not, then either the physics or the optimisation set-up is incorrect.

  • Check that the difference plots are indeed improving.

If the optimisation is to be re-run to refine the optimisation results, the following things should be considered:

  • Is the weighting of the different ramps in the cycle correct?
    (click here to go to the weighted script)

  • Is the physics correct?

  • Are the factor limits sensible?

When iSIGHT has completed its run successfully, with errors of ±15K transiently and ±5K at steady-state conditions, you have completed the matching exercise.

Discuss any issues with your training supervisor.

You should now be equipped to use iSIGHT for the matching of a real thermal model.

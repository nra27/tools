Exercise 7 - Thermal matching using SOFT

Introduction

SOFT is a graphical user interface that facilitates the integration of different engineering analysis codes, provides access to a range of optimisation algorithms and offers different visualisation tools to aid designers reduce design time whilst improving the quality of their final design.

This example is to show the user how to use SOFT to match a basic thermal model. From time to time throughout the tutorial, you will be referred to the SOFT Best Practice for more detailed information on a particular topic. 

Set up model

The method described below is recommended for ANY matching exercise, regardless of whether an optimiser is to be used.

Read the Best Practice page on setting up the sc03 model and then return to this page to continue with the exercise.

  1. In the SC03 general example base model, ensure that user parameters are added to the user variables as follows:

 

Name Expression
fcsh23 1
fcsh24 1
fcsh25 1
fcsh26 1
fcsh37 1
fcsh50 1
fcsh61 1
fcsh62 1
fcvh05 1
fcdh08 1

 

  1. Define the heat transfer expressions in the following boundary conditions as shown. These are the factors which are to be varied as part of the matching exercise with the additional term to represent the factor.

 

Boundary Condition

New HTC Expression
stream 23 fcsh23*FRD(NH)
stream 24 fcsh24*FRD(NH)
stream 25 fcsh25*FRD(NH)
stream 26 fcsh26*FRD(NH)
stream 37 fcsh37*FRD(NH-NI)
stream 50 fcsh50*MFCD(2250.3/102,5.3,17.6,SM50/102)
stream 61 fcsh61*FRD(NH)
stream 62 fcsh62*FRD(NH)
void 5.1 fcvh05*NVP(190)
duct 8 fcdh08*FRD(NH-NI)

 

  1. Produce key values in SC03 which record the difference between measured and predicted data.  

e.g.       DIFFP1 = tk@P1-GRP1(t)

where    GRP1(t) is the measured data at that particular point
            tk@P1 is the predicted metal temperature at position P1

  1. Save the model (File/Save file/Database)

 

Specify design parameters

The following steps describe how to perform an automatic thermal matching exercise using SOFT:

  1. To get the latest version of SOFT, in your UNIX workstation type:

ss01 -td .  

Note that the . is important!

On the main SOFT graphical user interface, click on File/New and specify a suitable name for the project (e.g. softest3.tmp). Then click OK.

  1. Specify the number of variables (10, for this thermal matching exercise) and click Assign.

Specify the variable names (fcsh23, fcsh24, fcsh25, etc), followed by the equals sign (=). This is needed to ensure compatibility between the SOFT and SC03 output and input files. Then click on Next.

  1. Specify the initial value for the parameters and the parameter upper and lower limits (see figure below). Ensure the selection boxes in the right hand side of the panel are turned on to include these parameters in the optimisation process. Click on Next.

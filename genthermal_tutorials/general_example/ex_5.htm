Exercise 5 - Thermal Matching

The complexity of convective heat transfer problems in gas turbine engines means that purely numerical solutions are not yet practicable. It is therefore necessary to validate temperature predictions against engine test data

Assume now that an engine test has just been completed. The test data has been stored on ALICE turned into .meas data using VW34.

  • The measured data will already be in the original model as graphical data items.

Exercise Objectives

To calibrate the finite element thermal model against measured engine test data, by changing the heat transfer coefficients in the thermal boundary conditions until an acceptable accuracy is reached.

The standard accuracies used are as follows

Type of measurement 

 Accuracy required
Transient  ±15K
Steady state   ±5K  

However, this example will give a match with an accuracy of ±1°K.

Problem

Given the basic model originally picked up at the start of exercise 1 (example.pm), compare the results at the following positions with measured data (GRP1 to GRP7) then modify relevant boundary conditions to achieve an acceptable match (±1K):

 

Position

 X Y
1 171.999  250.202
2 167.234 191.728
3 123.999 80.110
4 183.931 60.000
5 243.999 78.520
6 196.591 214.624
7 184.885 279.919

 

Instructions

  1.  Put reference points on the model at the co-ordinates specified above.

The geometry should be broken at the coordinates above (if not already done so), chained, meshed, and then use 'Snap to Nearest Node' to apply the reference points at the appropriate coordinates

Click here to get help with reference points

 

  1. Create difference plots to show difference between predicted and measured data (convention is predicted-measured)

Click here to learn about keyvalues

Click here to learn how to plot graphs quickly

Click here for help setting up the graph definition file

 

  1. Use engineering judgement to determine how to vary the boundary conditions to bring the difference between the measured and the predicted data down to within ±5K at steady state conditions and ±15K for transient conditions.

Click here for help to decide how/why to vary the boundary conditions

Click here for read the Thermal Matching validation notes

As this is a staged example, the difference plots can show a 0K difference

 

  1. When the difference plots give an accuracy of  ±1K, use the plugin SC77 (BC-Compare) to check how the boundary conditions have changed.

BC Compare is a function which compares the thermal boundary conditions from two separate models with one another and prints out any differences into the output window, a named .prt file and a text editor window.

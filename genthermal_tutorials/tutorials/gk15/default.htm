GK15 (SC03 Input Processor) Interactive Tutorial

Menu
Aims
Module Objectives
1) Introduction to SC03 Software
2) Loading the ALICE performance data
3) Linking ALICE data to a cycle definition
4)  Saving the ALICE performance data in a BDD file

 

  • You can open any of the sections in the training module by clicking the titles to the left of the picture above.

  • It is essential that you look at the Module Objectives before you begin working on the training content.
    After looking at the Module Objectives, work through the main sections of the module in the sequence indicated by the section numbers, beginning with Section 1: Introduction to SC03 Software. A test cycle is progressively developed as you work through the main sections of the module.

  • ALICE Performance Data Input forms part of the SC03 Thermal Analysis Training System. 

For this tutorial it is essential that you have access to the database where the test data is stored. This varies depending on the site where you are working. For Bristol users the database is THERMALB. For Derby the database is THERMAL. If you are not a member of the necessary user group, contact EDS to be included.

Note that the data contained within this tutorial are  currently only available on the Bristol THERMALB database but should be available in Derby shortly.

Aims

The overall aim of this training module is to enable users of SC03 to acquire the knowledge and skills necessary to read in ALICE performance data with the GK15 plugin.

Module Objectives

When you have worked through all of the sections contained in the module you will be able to identify and describe the main stages in the processing of the ALICE performance data with the GK15 plugin within the SC03 environment.

Specific sections of the module will enable you to:

  • explain how to load the GK15 plugin

  • explain how program options and commands are accessed in GK15

  • list the steps undertaken in the data accessing process

  • explain how to process cycle definition and performance data into a .bdd file

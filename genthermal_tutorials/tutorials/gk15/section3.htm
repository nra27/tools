GK15 (SC03 Input Processor) Interactive Tutorial

Section 3: Linking ALICE data to a cycle definition

Objectives of this Section

When you have completed this section you will be able to:

  • Create an analysis cycle

  • Link test extract to cycle condition

 

Learning Points of this Section

  1. Create analysis cycle

  2. Define base condition

  3. Define ALICE cycle definition

  4. Check BDD data

Click on one of the learning points above to find specific information or scroll down the page to begin the section

 

Create Analysis Cycle

In order to be able to link different test extracts to different analysis conditions, it is first necessary to create an analysis cycle. This is done in the Analysis/Cycle Definition window

For this tutorial we will create a square cycle based on three different performance conditions. 

Create a cycle with three condition numbers as follows:

Click More to be able to put in the last time step.

Click OK.

 

Define Base Condition

The base conditions are the start conditions of the model and are usually the ambient conditions. Since SC03 does not work well with zero speeds and flows, the base conditions include setting all flows and speeds to very small values.

Click on Model/Global Control for the Global Control window

Click on Thermal Ctrl and fill in the Base Parameters list with the following values.

Click OK in both the Thermal Analysis Control and in the Global Control windows.

 

Define ALICE Cycle Definition

For the definition of the run conditions, click on Analysis/ALICE Cycle Definition.

In the ALICE Cycle Definition window, the test is already listed for all chosen condition numbers but the data are not loaded.

Click on BASE/Load to choose the extract for the first condition number. The Base Condition data will be loaded for condition number 1.

Click on ALICE/Load to choose the extract for the other two condition numbers. The Extract Selection window will return and allow you to choose an extract.

Choose extract 0002000 for condition 2, and extract 0002001 for condition 3. The ALICE Cycle Definition window will now look as follows:

If you have done it wrong, you can click on Unload and redo your choice.

Click OK. The performance data is now loaded.

The Check BDD Data window will appear. In this window the data is checked in order to make sure that it is has not been corrupted during the transfer. You can edit the pre-set ranges to values of your choice.

Click OK. You will get up a warning that some data are out of range.

Click OK and go to the SC03 output window.

The parameters in question are listed for all conditions where they are out of range. In this case it does not matter, since the model does not use T11, T12, T13, PWSD or XNL. The speeds NG, NGHPT and NGLPT are between 3000 and 4000 rpm for both running conditions and it is OK for this case. 

Go back to the SC03 graphics window.

 

Check BDD data

You can do the same check at any time by choosing Analysis/Check BDD data in the menu.

The BDD data can be tabulated by returning to Analysis/Cycle Definition.  There are now values in both the Time History and the Environment Parameters tabs.

The Time History tab shows the analysis cycle that was defined when loading in the ALICE performance data.

The Environment Parameters tab shows the parameters that were defined in the Parameter Name Selection window during the reading of the ALICE performance data. Condition No 1 represent the base parameters as defined earlier.

Click on the up and down arrows to the right of the condition number to see the other cycle conditions. Click on More and Back to move through the parameter list. Click OK to close the window.

It is also recommended that you check your data via the Graphs/Plot Env. Parameters option.

In the parameters list, choose NG, P3 and T3.

Click OK and verify the values in the graphs.

All values seem reasonable. 

Close the graph and proceed to the next section for saving the BDD file.

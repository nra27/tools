3D Tutorial using Parametric Geometry and Tags

The 3D model below is basically the same as that created within the 3D tutorial for new users. The main changes for this tutorial are as follows:

  • the underlying geometry was produced in Unigraphics as a parametric model and passed to SC03 as a STEP file

  • the geometry is tagged (i.e. labelled) which allows boundary conditions to be applied via tags rather than by cursor detects

  • additional local mesh control has been applied to produce a more uniform mesh

Here is the scenario. A designer has provided you with geometry from which you have produced an SC03 thermal model to provide metal temperatures. The designer has now given you a second version of the geometry where the pitch circle diameter for the set of holes on the disc has been changed from 520mm to 540mm (i.e. the hole is now positioned further outboard on the disc). 

Old geometry

New geometry

The following steps describe how to save data from the original model and then apply it to the new geometry in an automated way.

The files required are available in the 3dtut_tag.zip file. This needs to be downloaded and then unzipped.

Note - when downloading files from the web to Unix there may be a requirement to remove extraneous characters using the dos2unix command (i.e. dos2unix webfilename unixfilename)

Using the original model

  • Download the original model (3dtut.pm) on to the Unix system (use 'save target as')

  • Open SC03 and read the model in (File/Open/Database)

  • Save the bdd file (File/Save/BDD) called tutorial2d.span.bdd

  • Save the genbp file (File/Save/GenBP) - this contains information on the boundary conditions and other model info

Note that the item GBPITEMS can be produced to save items such as LIMITS, TITLE, ANALTYPE, materials items etc if required.

In the new model

  • Download the new geometry (db_fe.v3.stp) on to the Unix system (use 'save target as')

  • Open SC03 and read in the step file (File/Open/STEP)

  • Read in the BDD file (File/Open/BDD) file called tutorial2d.span.bdd

  • Read in the genbp file (File/Open/GenBP) this is the file saved from original model

  • Mesh the Model (Mesh/Generate Mesh) Just OK the panel - the mesh settings from the original model have been copied across within the genbp file

  • Run the Model (Analysis/Run)

  • Save the database (File/Save/Database) to a new name

  • Post-process as required

 

Automation of the process

Note that the whole process of adding the information to the new model can be done via exec file.

By downloading all the files in the zip file above to the same directory (i.e. db_fe.v3.stp, tutorial2d.span.bdd, 3dtut.gbp, 3dtut.exec) the following can be done.

The exec file completes all the stages above automatically. It then performs some simple post processing - automatically displaying the metal temperatures at the stabilised max-take-off point in the cycle and produces a print file (3dtut.png) that could be used within WORD, etc.

To run the exec file select it from (Analysis/Exec File Control/Replay) from within SC03.

Alternatively run the whole thing from a Unix window by typing: sc03 -e 3dtut.exec

Exec File Listing

# Exec file to create the 3d tutorial STEP via tags
#
# Version 1
# Chris Barnes
# 15/6/2004
#
# Read in STEP file
rd step db_fe.v3.stp
# Read in the bdd file
rd bdd tutorial2d.span.bdd
# Read in the genbp file
# Note that this is the file written from previous model
# But with STRCNTL and THRCNTL removed (refer to expressions)
rd gbp 3dtut.gbp
# Mesh the geometry
mesh
# Call a second exec file to set global control to thermal
#include "3dtut.global.exec"
# run the model
call scf
# ensure that the default result file name is used
openver
rd fem sc03.jobid.fem
index
# Plot metal temperatures at max take off
case static 5 1 0
ctp tk
# make isometric view and 'find', then print
sview 7
locate
sdump 3dtut.png
# Write out final database
wr pm 3dtut.final.pm

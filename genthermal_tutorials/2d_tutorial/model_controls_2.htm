Domain Control

The Domain Control panel is used to define all properties for a domain.

The Domain label (top left on the panel) is actually a push button. When this button is pressed using the mouse controls, the control panel will clear and you can then detect a domain on any part of the model.

The outline of that domain is then highlighted (if geometry is present) and the control panel returns with the specified domain number filled in.

Select the Domain button and click on the shaft geometry. The shaft is now highlighted as Domain 1.

 

Repeat the domain selection process (described above) for the Disc using Domains 2, 3, & 4 (you will find that the disc has three domains, the region next to the shaft, the cooling hole and the outer surface).

 

Did you know...?

All materials used in Rolls-Royce are identified by an alpha-numeric code (generally a three-letter code). The first letter indicates the type of material (e.g. materials beginning with A are steels; those beginning with T are titanium alloys).

For more details, click here.

 

Now choose Domain 1 (shaft), and select the GET button. An Available Materials panel will now appear.

Select the COMMIT Database option. A Materials on COMMIT panel will appear.

Use the mouse to select the materials AEF and TDX. If you choose the wrong material by mistake, select UNDO.

Now select OK on the Materials on COMMIT panel. When the Confirm Material Symmetry panel appears choose the Isotropic option for both materials, and select OK.

Using the GET button, select material property TDX for Domain 1, and material property AEF for Domains 2, 3, & 4.

Now complete the Domain Control panel for all four domains as follows:

Did you know...?

The representation of 3D features within a 2D axisymmetric model is generally performed by defining those regions as Plane Stress with an appropriate thickness property. Note that the thickness expression will only be used by SC03 if Plane Stress (i.e. not Axisymmetric) is selected as the domain type.

 

Also on the control panel there are a number of headings. e.g. Orientation, Thickness, Thermal Loads, Structural Loads, Copy changes to all, and Copy domain to all.

 

Click on the heading Thickness for Domain 3.

Select the expression tab, and enter the thickness as 1256mm.  (This is the thickness of metal which isn't hole in this domain)

Input Checking

All thermal boundary conditions for the problem that we have been tackling in this training module have now been defined. 

Because of the large number of expressions used in thermal analysis it is recommended that all input data is checked before analysis. 

In addition it is essential that all calculated boundary condition data is checked following an analysis.

For further information see Model input checking best practice 

The following facilities exist in SC03 to aid the checking of analysis models.

  • Query

  • Thermal boundary condition display

  • Summary print

The next three sections explain each of these facilities in more detail. 

Input Checking: Query

The model can be manipulated within any command. There is no need to end the Query command to move or zoom in/out of the model.

Boundary Conditions

Select Model/Query BC with the left mouse button to get the boundary condition query arrow.

By clicking on the boundary conditions, the current value of any boundary condition(s) on a surface is reported.

The boundary condition can then be edited to correct any input errors.

Geometry

The 2D Geometry Query command is used to report the current geometrical properties of any line on the model.

If the 2D Geometry Commands box is not already open, first open the Model Menu by selecting Model with the left mouse button and then select the 2D Geometry Commands option.

Then select the Query command from inside the box and check the thermal boundary conditions previously defined.

 

Input Checking: Thermal Boundary Condition Display

On complex models, it is often difficult to determine where individual thermal boundary conditions have been assigned. The Display/Property Display/Thermal BC tab option allows single or multiple boundary conditions of a particular type to be colour highlighted or displayed using symbols.

Select the Display Menu and Property Display as before, then select the Thermal BC tab on the panel.

                 

Set the Thermal BC controls to those shown above and move the model to see how a single boundary condition is highlighted.

 

  • The boundary condition symbol size is controlled by the Line Contour Label Size. This is changed from the Display Settings/Miscellaneous option on the Display Menu.

 

Input Checking: Summary Print

While the Query command is useful for checking and editing individual boundary conditions, it is not suitable for completing a thorough check of all input data.

This can be achieved by generating a summary print (.prt) file.

Open the File Menu by selecting File with the left mouse button.

 

 

Then select the Save File command with the left mouse button and click the Summary tab on the Save File panel. Enter a summary file name in the panel (ending with .prt) and click OK.

 

 

 

 

In the Summary File Contents panel select the All option for the Physics Model.

 

 

 

 

 

 

 

          

A listing of all boundary conditions expressions will be written to the .prt file, and a window opened displaying the contents of the file.

  • The summary print file contents depend on what is contained within the current SC03 session. We will use the summary print option later to generate a summary of calculated boundary condition values, when results have been evaluated.

 

Check that all thermal boundary condition expressions are correctly defined.

SC03 3D Thermal Modelling

Section 1: Introduction to SC03 Software

Objectives of this Section

When you have completed this section you will be able to:

  • identify the main program options that are controlled from the SC03 window

  • explain how the main menus and commands are accessed in SC03

  • give examples of the use of keyboard shortcuts in controlling display settings

Learning Points of this Section

  1. Starting SC03 

  2. Using the Mouse

  3. Mouse Button Usage Summary

  4. Keyboard Shortcuts

  5. Display Settings

 

Starting SC03

The first step is to start the SC03 software.

If SC03 is already running on your machine, follow this link.

There are several different methods for starting SC03, depending on the type of machine being used and the way in which your operating system has been configured. The recommended method for starting SC03 when you are using it with the training system is to:

Type sc03 in a shell window

The previous command assumes you have a path to the SC03 script directory defined in your login files.

Type /usr/rreng/script/sc03 in a shell window

On the Technical PC, use the SC03 icon on the desktop to start SC03.

When SC03 has started, your display should look similar to this:

The main SC03 display window is used to enter commands, control the program display, and enable visualisation of your SC03 model.

The output window is used by SC03 to give messages to the user.

All SC03 windows may be re-sized to suit individual preferences but it is recommended that the output window is always kept visible. This will allow any program warnings or errors to be seen. 

Program errors are also highlighted in the status bar at the bottom of the SC03 display window.

Using the Mouse

Most commands can be accessed from the pull-down menus, which are operated by the left mouse button.

You can also operate menus from the keyboard, by pressing the Alt key with the menu mnemonic (underlined character). When the menu is displayed, press the mnemonic for the desired command (this time without the Alt key).

Pull-down menus can be 'torn off' by clicking on the dotted line (left mouse button), and can then be positioned as required. They will then remain visible until you close them down, either from the window manager or by pressing the Escape key.

The right mouse button brings up a context-sensitive popup menu. Over the graphics window this gives quick access to the last command and the Screen commands. With the Control key held down, it displays the last 10 commands.

Mouse Button Usage Summary

A summary of how the different mouse buttons are used with SC03 is given below.

 

Key Left Middle Right
- - Pan Screen Commands
Shift Window Zoom Last 10 Commands
Ctrl - Rotate X & Y (3D Only) -
Alt - Rotate Z (3D Only) -

 

Note that on very rare occasions, some uses of the mouse can cause the SC03 session to 'lock-up'. The only solution is to log onto another machine and kill the SC03 session. This will cause all unsaved changes to be lost. To avoid unnecessary loss, it is important to save your model often. (e.g. use two different file names and save alternately to them.) Click here to go the Best Practice to explain how to kill an SC03 session from another machine. It is suggested that this Best Practice is printed out.

  • If you want to learn more about the mouse now, go to the SC03 Users Guide (Section 4.11 Viewing the Model)

 

Keyboard Shortcuts

All the display options - which control what is actually displayed on screen - can be accessed:

  • from the Display Settings panel or

  • from the Display menu or

  • by pressing the corresponding letter key.

Full details of keyboard usage is provided in the SC03 Training Section: 'Learn about SC03 Commands'. 

  • If you want to learn more about the keyboard now, go to the SC03 Users Guide (Section 4.9)

 

Display Settings

The Display Settings panel is an important command that is used repeatedly to control the SC03 display:

The single letter shown can be typed directly into the display window to toggle the display of what the letter represents. In some cases extra detail is shown by typing Shift plus the letter:

Shift-A animates static results
Shift-G geometry entity numbers
Shift-H shadows behind the model instead of underneath
Shift-N nodes drawn as circles instead of numbers
Shift-W shrunken elements
Shift-X annotated axes with global origin shown by sphere
Shift-Y multi-face BC local axes arrows

Domain Control

The Domain Control panel is used to define all properties for a domain.

The domain label (top left on the panel) is actually a push button. When this button is pressed using the mouse controls, the control panel will clear and you can then detect a domain on any part of the model.

The outline of that domain is then highlighted (if geometry is present) and the control panel returns with the specified domain number filled in.

Select the Domain button and click on the inner shaft of the tutorial problem. The shaft is now highlighted as Domain 1.

The panel will now show Domain 2; again select on the Domain button and click on the outer disc/shaft to define this as Domain 2.

Suppress mesh refinement

By default SC03 will attempt to refine the mesh to achieve the requested accuracy. We must therefore suppress all mesh refinement.

Open the Mesh Menu by selecting Mesh then select Suppress Refinement then select the Block... option

When the + cursor appears define a block/window that fully encompasses the analysis model by holding the left mouse button and dragging out a rectangle; this becomes a block with depth perpendicular to the screen covering the extent of the model.

Input checking

All thermal boundary conditions for the problem that we have been tackling in this training module have now been defined.

Due to the large number of expressions used in Thermal Analysis it is recommended that all input data is checked before analysis.

In addition it is essential that all calculated boundary condition data is checked following an analysis.

For further information, see  Best Practice on Model Input Checking 

The following facilities exist in SC03 to aid the checking of analysis models.

  • Query

  • Thermal Boundary Condition Display

  • Summary Print

The next three parts below explain each of these facilities in more detail. scroll down or click on one of the links above for information on a particular item.

Input checking - Query

The Query command is used to report the current value and allow editing of any boundary conditions on a surface.

 

Select the Query BC command from the Model Menu inside the box and check the thermal boundary conditions previously defined.

 

The model can be manipulated within any command. There is no need to end the Query command to move or zoom in/out of the model.

 

Input checking - Thermal Boundary Condition Display

On complex models, it is often difficult to determine where individual thermal boundary conditions have been assigned. The Display/Property Display/Therm BC tab option allows single or multiple boundary conditions of a particular type to be colour highlighted or displayed using symbols.

Select the 'Display Menu' and Property Display as before, then select the Thermal BC (Therm BC) tab on the panel.

Set the Thermal BC controls to those shown above and move the model to see how a single boundary condition is highlighted.

 

The boundary condition symbol size is controlled by the Line Contour Label Size. This is changed from the Display Settings/ Miscellaneous option on the Display Menu.

 

Input checking - Summary Print

While the Query command is useful for checking and editing individual boundary conditions, it is not suitable for completing a thorough check of all input data.

This can be achieved by generating a summary print (.prt) file.

Open the 'File Menu' by selecting 'File' with the left mouse button

 

then Select the Save File command with the left mouse button and click the Summary Tab on the Save File panel. Enter a summary file name in the panel (ending with .prt) and click OK.

 

 

In the Summary File Contents panel select the All option for the Physics Model.

 

 

 

 

A listing of all boundary conditions expressions will be written to the .prt file, and a window opened displaying the contents of the file.

The Summary Print file contents depends on what is contained within the current SC03 session. We will use the Summary Print option later to generate a summary of calculated boundary condition values, when results have been evaluated.

 

Check all thermal boundary condition expressions are correctly defined.

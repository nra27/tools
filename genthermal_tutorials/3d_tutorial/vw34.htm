Starting VW34

Like SC03, vw34 may be started by a number of different methods dependent on the type of machine being used and how your operating system has been configured.

    • GK15 has now superseded VW34 for SC03 input processing except for cycle definition (bdd) creation from R280 BDD data.

The recommended method for starting vw34 for use with the training system is to:

Type vw34 in a shell window

 

 

This command assumes you have a path to the vw34 script directory defined in your login files. If vw34 does not start then:

Type /usr/rreng/script/vw34 in a shell window

 

 

VW34 introductory screen

When VW34 has started, your display should look similar to this:

The main VW34 display window provides access to the main commands and shows program messages.

The Program Control panel enables key items to be created and written to the BDD file.

    • Note that VW34 retrieves data from the mainframe R280 system, and will therefore request your ETSO mainframe password.

 

Cycle Definition in VW34

Select the Cycle Definition option on the VW34 Program Control Panel

Usually, the cycle definition (i.e. the engine operating conditions) and the associated time spent at each condition will be known. The types of cycles often considered in thermo-mechanical analysis of aero engines are described in the Temperature FE Analysis process (formerly MMM26200 Problem Definition & Management).

For further information see:  Engine Cycle Definitions Best Practice

On selecting the Cycle Definition option, a sub-panel appears, prompting you  for the method to be used for defining performance data.

 

 

 

Manual Input:

 = All data to be entered manually
R280 BDD Link (Book 63) = Data to be retrieved from a mainframe R280 BDD book
ALICE Link (Test Data) = Data to be retrieved from a mainframe ALICE database

 

Select the R280 BDD Link option

 

    • You may now be asked to login to the mainframe.  If you do not have permission to access the mainframe system then consult your training supervisor.

 

The Flight/Test Cycle Time and Performance panel should now be displayed.

 

 

 

 

 

 

 

 

The bp (base parameters) option defines the ambient conditions.

Select the Base Parameters option on the Program Control Panel.

This opens the Base Parameter Values panel.

 

Enter the ambient conditions in the appropriate units:

 

Initial Pressure = 101250 Pa
Initial Temperature = 288.15 K
Initial Mass Flow = 1E-6 kg/s  
Speed below which Base Params. apply = 1 rad/s

 

    • The Initial Mass Flow has to be set to a small number as SC03 may experience problems with a zero flow.

 

Now click OK to complete definition of the Base Parameters data.

 

Complete the cycle definition using the following data (you will have to use the More button to complete the table):

Book: 63 Chapter: 58721109

This is a standard 3000 sec. square cycle.

 

    • The description of each time is not currently used by SC03.

    • Use the More/Back buttons to complete the cycle definition.

    • The BDD data on this chapter are in Imperial Units

Select OK in the Cycle Definition window

VW34 will then attempt to access the performance information requested from the mainframe computer.  Refer to the main VW34 display window, which should return the following messages:

Reading names verse for chapter 58721109...
Reading verse 16, Chapter 58721109
Reading verse 1, Chapter 58721109
Checking data - please wait ...
Flight cycle definition complete

It is worth noting that the R280 BDD link cycle definition panel also includes the following additional capabilities:

 

Control File:- This enables an existing .bdd file to be selected and all or some of the associated time points/verses and chapters to be retrieved.
Global Change:- Allows all references to a chapter to be updated
Entire Chapter:- Causes all verses on a chapter to be added to the .bdd file, even if they are not referenced in the cycle definition.

At this point, save the file  by clicking on the Write BDD Data button and saving the file with a .bdd extension.

Section 3: Cycle Definition

Objectives of this Section

Using scaling expressions to describe boundary conditions

The thermal or thermo-mechanical SC03 model will incorporate a set of boundary conditions that define the fluid conditions or applied loads at the component surfaces.

Rather than assign absolute values of temperature, pressure or load etc., which would limit the model to solving for a single engine operating condition only, it is recommended that expressions are defined in terms of main engine performance parameters (P26, T26, W26, NH etc.) for all boundary conditions. These parameters refer to defined points in the engine.

     

For further information see: Pressure & Temperature Scaling Best Practice 

The use of scaling expressions means that the thermo-mechanical model is independent of engine condition. The engine performance/cycle definition may therefore be changed and the same analysis model can be used to solve any transient or steady state problem.

The engine performance and cycle definition can be entered directly in SC03, but for large models or complex cycles an interface to the R280 BDD (Basic Design Data) system or ALICE database is provided via VW34 and the use of a bdd file.

VW34 also includes links to the Cooling Air Network system SPAN. This is used to automatically retrieve and/or scale SPAN pressures and flows for input into SC03. 

More recently, GK15 has been developed which can be used to create a cycle definition from ALICE data.  It can also be used to incorporate SPAN data.

Reading a .bdd file into SC03

The .bdd file must now be read into SC03.

  • If you have skipped the vw34 section, the completed bdd file can be found at 
    /usr/rreng/data/sc77/tutorial2d.span.bdd

  • If you cannot access this directory, the required files are stored on the web.

Return to your SC03 session and:

Open the File Menu by selecting File with the left mouse button

then select the Open File command with the left mouse button

 

 

 

 

 

 

At the top of the selector panel are two rows of tabs

 

 

 

 

 

 

 

Select the BDD tab with the left mouse button and type the .bdd file name in the file name box or use the Up button and directory listings to move to the appropriate directory, and select the file using the left mouse button.

Click the OK button to read the .bdd file into SC03

 

Cycle Definition & Control in SC03

The cycle definition and performance data read from the .bdd file can now be displayed and, if necessary, modified.

Open the Analysis Menu by selecting Analysis with the left mouse button

 

 

 

then select the Cycle Definition command

 

 

 

 

 

The tabbed Cycle Definition panel should now be displayed:

Time History Tab...
contains the cycle definition read from the  .bdd file

Environment Parameters Tab...
contains environment parameters (the engine performance data) also from the .bdd file.

Both the cycle definition and performance data can be viewed and edited as required.

Practice viewing the cycle definition and environment parameter data.

 

  • Engine condition numbers are re-numbered sequentially when a .bdd file is read.

  • If a simple engine cycle with limited engine performance data is required for the analysis, all cycle times and environment parameter names and values can be defined using these panels, without using VW34.

Environmental parameters can also be viewed graphically (against time).

Open the Graphs Menu by selecting Graphs with the left mouse button

Then select the Plot Env. Parameters command. Click the OK button to read the .bdd file into SC03

 

 

 

 

 

then select the parameter that you require to be plotted against time (e.g. PAMB), and finally click on OK to generate the graph.
